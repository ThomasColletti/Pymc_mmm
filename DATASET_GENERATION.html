Step 2: Saturation (Diminishing Returns)

Logistic function models how ROI decreases with increased spend:

Hill Saturation Function:
y = x^s / (x^s + K^s)
  • K = Half-saturation point (spend level for 50% max effect)
  • s = Shape parameter (elasticity)
Channel K (rel. to avg) s (shape) Rationale
Search 0.5√ó 1.0 Saturation hits fast; limited online audience pools
Social 0.8√ó 1.2 Moderate saturation; audience fatigue effects

TV 2.0√ó 2.0 Massive reach; minimal saturation within budget ranges
üìà Real-World Principle

Lower-funnel channels (Search) exhaust addressable inventory faster. Broadcast channels (TV) scale more efficiently across broader audiences. This is precisely captured by the K parameter.

8. Technical Implementation

Architecture

  1. Generate Spend: Gamma distributions per channel
  2. Apply Adstock: Geometric carryover with channel-specific alphas
  3. Normalize: Scale adstocked spend to zero mean, unit variance
  4. Apply Saturation: Logistic function with K and s parameters
  5. Combine Media: Sum weighted contributions across channels
  6. Add Controls: Trend, seasonality, promotional events
  7. Add Noise: Normal(0, 12) for realism
  8. Export: Format to CSV for MMM training

Configuration Parameters

All assumptions are explicitly coded and tunable:

configs = { 'search': {'alpha': 0.05, 'lam': 2, 'beta': 8}, 'social': {'alpha': 0.2, 'lam': 3, 'beta': 12}, 'ctv': {'alpha': 0.5, 'lam': 4, 'beta': 15}, 'tv': {'alpha': 0.8, 'lam': 5, 'beta': 25} }

This enables sensitivity analysis, scenario planning, and dataset customization for different real-world scenarios.
